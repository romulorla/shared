What is a Recipe?

A recipe stores all the settings needed to produce a specific pipe product — product name, dimensions (diameter, height, wall thickness), product type, and all machine parameters (speeds, positions, timings). The system stores up to 100 recipes in the PLC's permanent memory, preserved across power cycles.

Workflow 1 — Loading and Running a Recipe

The most common workflow: select a stored recipe, load it into the HMI, review, and send it to the machine.

Workflow 3 — Saving Machine Adjustments During Production

After the operator adjusts parameters during production, use Sync to pull runtime values, then save them permanently.

        graph TD
          A["Production adjustments\nmade on machine"]:::op --> B["Open Recipe screen"]:::op
          B --> C["Press LESEN\n(Read) if recipe\nnot loaded"]:::read
          C --> D["Press SYNC\n(Pull machine values)"]:::sync
          D --> E["Review synced\nparameters"]:::op
          E --> F{"Values\nacceptable?"}:::op
          F -->|Yes| G["Press SPEICHERN\n(Save)"]:::write
          F -->|No| H["Make further\nadjustments on\nmodule screens"]:::op
          H --> D
          G --> I["Updated recipe\nstored permanently ✓"]:::done

⚠ Remember: If you skip the Speichern step, your runtime adjustments exist only in the HMI's temporary memory. A power loss or recipe reload will discard them.

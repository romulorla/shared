Recipe System — Operator Guide

How to create, edit, store, and load product recipes from the HMI panel

Author: Romulo Lange Date: March 2026 Platform: Siemens S7-1500 / TIA Portal Version: 1.0

What is a Recipe?

A recipe stores all the settings needed to produce a specific pipe product. This includes the product name, dimensions (diameter, height, wall thickness), product type, and all machine parameters (speeds, positions, timings, etc.).

The system can store up to 100 recipes. Recipes are saved in the PLC's permanent memory and are kept even after a power cycle.

The Recipe Screen

When you open the recipe screen on the HMI, you will see:

Recipe List (top area)

A table showing all stored recipes with the following columns:

ColumnDescription
Program IDUnique number identifying the recipe (1–100)
NameProduct name
NennweiteNominal diameter
HöheProduct height
WandstärkeWall thickness
ProdukttypProduct type

You can select a recipe by tapping on its row in the list.

Search and Filters (top bar)

  • Suchen (Search): Type a name to search for a specific recipe.
  • Nennweite / Höhe / Produkttyp dropdown filters: Filter the recipe list by diameter, height, or product type.

Action Buttons (bottom bar)

The buttons at the bottom of the screen perform the following actions. Each button and its function is explained in detail below.

Button Reference

Lesen (Read)

What it does: Loads the selected recipe from storage into the HMI so you can view and edit it.

How to use:

  1. Select a recipe from the list by tapping its row.
  2. Press Lesen.
  3. The recipe parameters are now loaded and visible on the HMI.

When to use: When you want to view a stored recipe's full parameters, or before downloading it to the machine.

Erstellen (Create)

What it does: Creates a new recipe and saves it to storage.

How to use:

  1. Press Erstellen.
  2. A dialog will appear asking for the new recipe details:
    • Program ID (pick an unused number 1–100)
    • Product name
    • Product type
    • Diameter, height, and wall thickness
  3. Confirm to create the recipe.
  4. The new recipe appears in the list.

When to use: When you need to set up a recipe for a new product.

Bearbeiten (Edit)

What it does: Modifies the product information of an existing recipe (name, type, dimensions).

How to use:

  1. Select the recipe you want to edit from the list.
  2. Press Bearbeiten.
  3. A dialog will appear with the current values. Change what you need.
  4. Confirm to apply the changes.

When to use: When product information (name, diameter, height, wall thickness, or type) needs to be corrected or updated.

Note: This edits the product identity information only. To change machine parameters (speeds, positions, timings), load the recipe first with Lesen, adjust the parameters on the respective module screens, and then save with Speichern.

Speichern (Save)

What it does: Saves the currently loaded recipe (including any parameter changes) back to its storage slot.

How to use:

  1. Make sure the recipe you want to save is currently loaded (use Lesen first if needed).
  2. Adjust any parameters on the module screens as needed.
  3. Return to the recipe screen and press Speichern.
  4. The recipe is saved to storage.

When to use: After making parameter changes that you want to keep permanently.

Important: If you change parameters but do not press Speichern, your changes will be lost if the recipe is reloaded or the system loses power.

Sp. unter (Save As)

What it does: Creates a copy of the selected recipe with a new Program ID and name. All machine parameters are duplicated to the new slot.

How to use:

  1. Select the recipe you want to copy from the list.
  2. Press Sp. unter.
  3. Enter a new Program ID (pick an unused number) and a new name.
  4. Confirm to create the copy.
  5. The new recipe appears in the list with all the same parameters as the original.

When to use: When you want to create a variation of an existing recipe without modifying the original (e.g., same product but with slightly different parameters).

Sync

What it does: Updates the loaded HMI recipe with the current machine values. The product information (name, type, dimensions) stays unchanged — only the machine parameters (speeds, positions, timings) are updated.

How to use:

  1. Make sure the correct recipe is loaded on the HMI (use Lesen if needed).
  2. Press Sync.
  3. The HMI recipe now reflects the actual values currently running on the machine.

When to use: When the machine has been running and parameters may have been adjusted during production. Use this to pull the current runtime values back into the HMI before saving.

Typical workflow: Sync → review changes → Speichern (to keep them permanently).

Laden (Download)

What it does: Sends the currently loaded HMI recipe to the machine. The machine will immediately begin using these parameters.

How to use:

  1. Load a recipe using Lesen.
  2. Review the parameters to make sure everything is correct.
  3. Press Laden.
  4. The recipe is now active on the machine.

When to use: When switching to a different product or when you want to apply a modified recipe to the machine.

Important: The machine will start using the new parameters immediately. Make sure the machine is in a safe state before downloading a new recipe.

Delete (Trash icon)

What it does: Permanently deletes the selected recipe from storage. All parameters are cleared.

How to use:

  1. Select the recipe you want to delete from the list.
  2. Press the trash icon button.
  3. The recipe is removed from storage and the list.

When to use: When a recipe is no longer needed and should be removed.

Warning: This action cannot be undone. The recipe and all its parameters will be permanently erased.

Refresh (Circular arrow icon)

What it does: Refreshes the recipe list display.

Print (Printer icon)

What it does: Prints the recipe information.

Help (? icon)

What it does: Opens the help information for the recipe screen.

Common Workflows

Loading and running a recipe

  1. Open the recipe screen.
  2. Find the recipe in the list (use the search bar or filters if needed).
  3. Select the recipe by tapping its row.
  4. Press Lesen (Read) to load it.
  5. Review the parameters.
  6. Press Laden (Download) to send it to the machine.

Creating a new recipe from scratch

  1. Press Erstellen (Create).
  2. Enter the product information (ID, name, type, dimensions).
  3. Confirm to create the recipe.
  4. Press Lesen (Read) to load the new recipe.
  5. Go to each module screen and configure the machine parameters.
  6. Return to the recipe screen and press Speichern (Save) to store all parameters.

Saving machine adjustments made during production

  1. After production adjustments are made on the machine, open the recipe screen.
  2. Make sure the correct recipe is loaded (press Lesen if needed).
  3. Press Sync to pull the current machine values into the HMI.
  4. Review the synced values.
  5. Press Speichern (Save) to permanently store the updated parameters.

Creating a variation of an existing recipe

  1. Select the original recipe from the list.
  2. Press Sp. unter (Save As).
  3. Enter a new Program ID and name for the copy.
  4. Confirm to create the copy.
  5. Press Lesen to load the new copy.
  6. Adjust the parameters as needed on the module screens.
  7. Press Speichern (Save) to store the changes.

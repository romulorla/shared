Read / View Actions
📖
Read READ
Lesen

Loads the selected recipe from storage into the HMI so you can view and edit its parameters.

  1. Select a recipe from the list by tapping its row
  2. Press Lesen
  3. Recipe parameters are now loaded and visible on the HMI

When to use: Before viewing a recipe's full parameters, or before downloading it to the machine.

Write / Create Actions
✚
Create WRITE
Erstellen

Creates a brand-new recipe and saves it to storage with the product identity you specify.

  1. Press Erstellen
  2. Enter Program ID (unused, 1–100), product name, type, and dimensions
  3. Confirm to create the recipe
  4. The new recipe appears in the list

When to use: Setting up a recipe for a new product.

✏️
Edit WRITE
Bearbeiten

Modifies the product identity information (name, type, dimensions) of an existing recipe.

  1. Select the recipe from the list
  2. Press Bearbeiten
  3. Change values in the dialog
  4. Confirm to apply changes

When to use: Correcting or updating product info (name, diameter, height, wall thickness, or type).

ℹ Note: This edits product identity only. To change machine parameters (speeds, positions, timings), use Lesen → adjust on module screens → Speichern.
💾
Save WRITE
Speichern

Saves the currently loaded recipe — including any parameter changes — back to its storage slot permanently.

  1. Ensure the recipe is loaded (use Lesen first if needed)
  2. Adjust parameters on module screens as needed
  3. Return to the recipe screen and press Speichern

When to use: After making parameter changes you want to keep permanently.

⚠ Important: Unsaved changes will be lost if the recipe is reloaded or the system loses power.
📋
Save As WRITE
Sp. unter

Creates a complete copy of the selected recipe with a new Program ID and name. All machine parameters are duplicated.

  1. Select the recipe you want to copy
  2. Press Sp. unter
  3. Enter a new Program ID and name
  4. Confirm to create the copy

When to use: Creating a variation of an existing recipe without modifying the original.

Sync & Transfer Actions
🔄
Sync SYNC
Sync

Pulls the current machine runtime values into the HMI recipe. Product identity stays unchanged — only machine parameters (speeds, positions, timings) are updated.

  1. Ensure the correct recipe is loaded (use Lesen if needed)
  2. Press Sync
  3. HMI now reflects current machine values

When to use: After production adjustments — pull runtime values before saving.

💡 Typical workflow: Sync → review changes → Speichern (to keep permanently).
⬇️
Download TRANSFER
Laden

Sends the currently loaded HMI recipe to the machine. The machine begins using these parameters immediately.

  1. Load a recipe using Lesen
  2. Review the parameters
  3. Press Laden
  4. Recipe is now active on the machine

When to use: Switching to a different product or applying a modified recipe.

⚠ Important: The machine starts using new parameters immediately. Ensure the machine is in a safe state before downloading.
Destructive Actions
🗑️
Delete DESTRUCTIVE
Löschen (Trash icon)

Permanently deletes the selected recipe from storage. All parameters are cleared.

  1. Select the recipe to delete from the list
  2. Press the trash icon button
  3. Recipe is removed from storage and the list

When to use: When a recipe is no longer needed and should be removed.

🛑 Warning: This action cannot be undone. The recipe and all its parameters will be permanently erased.
Utility Actions
🔃
Refresh UTILITY
Aktualisieren (↻ icon)

Refreshes the recipe list display to show the latest stored recipes.

🖨️
Print UTILITY
Drucken (🖨 icon)

Prints the recipe information for documentation or archival purposes.

❓
Help UTILITY
Hilfe (? icon)

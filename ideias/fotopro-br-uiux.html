React Native vs Flutter

A framework comparison specifically for an image-heavy, camera-dependent app targeting Brazilian marketplace sellers.

CriteriaReact NativeFlutterWinner
Camera Access react-native-camera / expo-camera ‚Äî mature, but relies on bridge to native. Occasional lag on older Android devices. camera plugin ‚Äî direct Dart access, smoother integration with preview stream. Better raw performance. Flutter
Image Processing Must bridge to native for Core Image / RenderScript. JS thread can bottleneck heavy image ops. Skia rendering engine is native. dart:ui for pixel manipulation. Can call platform channels for ML Kit / Vision. Flutter
UI Performance New Architecture (Fabric + TurboModules) improved significantly. Still JS thread dependency for complex UIs. Compiles to native ARM. 120fps capable. No bridge overhead. Ideal for image-heavy smooth scrolling. Flutter
App Store Deployment Mature. Well understood. OTA updates via CodePush. Mature. Slightly larger APK/IPA (~5-8MB overhead). Shorebird for OTA. Tie
Developer Ecosystem Larger community. More packages on npm. Easier to hire JS/TS devs in Brazil. Smaller but fast-growing. Google-backed. Dart is less common in BR market. React Native
Firebase / Google Cloud FlutterFire has first-class support but so does React Native Firebase. FlutterFire is Google's own ‚Äî tightest integration with Vertex AI, Cloud Storage, Firestore. Flutter
Offline / Local DB AsyncStorage, WatermelonDB, SQLite Hive, Isar, Drift (SQLite). More performant for large image metadata. Flutter
Animations Reanimated 3 is powerful but complex API. Built-in animation framework is elegant. Rive / Lottie integration seamless. Flutter

üèÜ Recommendation: Flutter

Flutter wins for FotoPro BR due to superior image processing performance, native Skia rendering (critical for smooth gallery/editor UX), first-class Google Cloud integration (our backend), and better camera plugin architecture. The Dart hiring challenge in Brazil is mitigated by the solo/small-team nature of the MVP. The compiled-to-ARM advantage directly impacts the core UX of an image-heavy app.

Processing UX

The processing screen is designed to feel fast and magical. Each step completes visually as it finishes on the backend, giving a sense of progress.

Steps

  • Step 1Background removal (on-device, ~500ms). Checkmark appears immediately.
  • Step 2AI enhancement via Gemini Flash API (1-3s). Pulsing indicator during upload + processing.
  • Step 3Final compositing + resize (~200ms local).

Details

  • Duration2-5 seconds total. Estimated time shown at bottom.
  • AnimationTriple concentric spinning rings (emerald, sun, ocean). Different speeds create organic motion.
  • Error StateIf API fails: "Algo deu errado. Tentar novamente?" with retry button. After 3 retries: fallback to on-device-only result.
  • Offline ModeIf no internet: on-device bg removal only. Toast: "Sem internet ‚Äî edi√ß√£o b√°sica aplicada".
  • HapticSuccess haptic (notification type) when processing completes.

Motion Design

Micro-interactions & Animations

Animations serve a purpose: they mask wait times, provide feedback, and create delight. All animations respect reduced motion settings.

Camera Shutter

Circular clip-path animation that closes to center and reopens. Duration: 300ms. Combined with medium haptic impact and a subtle "click" sound effect. White flash overlay at midpoint (50ms, opacity 0.3).

clip-path: circle(50%) ‚Üí circle(0%) ‚Üí circle(50%)
duration: 300ms ease-in-out
haptic: mediumImpact at 0ms
flash: white overlay, 50ms at 150ms

Background Reveal

Horizontal wipe that reveals the new background behind the product. Like pulling away a curtain. Left-to-right sweep with subtle blur at the edge.

type: horizontal mask sweep
duration: 600ms ease-out
direction: left ‚Üí right
edge effect: 8px gaussian blur

Processing Spinner

Triple concentric rings spinning at different speeds and directions. Creates organic, hypnotic motion that makes wait time feel shorter.

Export Celebration

üéâ

Confetti burst on successful export. 20-30 particles in brand colors. Duration: 1.5s. Combined with success haptic pattern. "Foto pronta! üéâ" text scales in with spring animation.

Animation Guidelines

  • DurationMicro: 150-200ms. Standard: 300ms. Complex: 500-600ms. Never exceed 800ms.
  • EasingDefault: ease-out (deceleration). Entrances: ease-out. Exits: ease-in. Springs: damping 0.8, stiffness 300.
  • Reduced MotionCheck `MediaQuery.disableAnimations` (Flutter) / `prefers-reduced-motion`. Replace animations with instant state changes.
  • PerformanceOnly animate `transform` and `opacity`. Never animate layout properties. Use `RepaintBoundary` in Flutter for complex animations.
  • Swipe MockupsHorizontal PageView with physics-based scrolling. Snap to center. Scale side items to 0.85. Parallax offset on background.
